# Mathematical Theorem Skill
# Provides guidance for formulating and presenting mathematical theorems

## Theorem Quality Checklist

When formulating a theorem, ensure:

- [ ] **Clear Statement**: The theorem is stated precisely without ambiguity
- [ ] **Defined Terms**: All technical terms are defined before use
- [ ] **Explicit Assumptions**: All conditions and hypotheses are stated
- [ ] **Novelty**: The theorem offers something new or generalizes existing work
- [ ] **Verifiability**: The theorem can be verified or disproved by counterexample

## Theorem Structure Template

```markdown
## Theorem: [Name/Number]

**Statement**: 
[Formal statement in mathematical notation]

**Variables**:
- $x$: [definition]
- $n$: [definition]

**Assumptions**:
1. [Assumption 1]
2. [Assumption 2]

**Claim**:
[What the theorem asserts]
```

## Proof Quality Checklist

- [ ] **Logical Flow**: Each step follows logically from previous steps
- [ ] **Justification**: Every inference is justified (cite theorems, axioms)
- [ ] **Complete Cases**: All cases are addressed (exhaustiveness)
- [ ] **Notation Consistency**: Same notation used throughout
- [ ] **Conclusion**: QED or □ marks the end clearly

## Proof Strategies

1. **Direct Proof**: Start with assumptions, derive conclusion
2. **Proof by Contradiction**: Assume ¬P, derive contradiction
3. **Proof by Induction**: Base case + inductive step
4. **Contrapositive**: Prove ¬Q → ¬P instead of P → Q
5. **Constructive**: Show existence by explicit construction
6. **Non-constructive**: Existence proof without construction

## LaTeX Tips

- Use `\mathbb{R}`, `\mathbb{N}` for number sets
- Use `\sum`, `\prod`, `\int` for operators
- Use `\lim`, `\sup`, `\inf` for analysis
- Use `\equiv` for equivalence relations
- Use `\Rightarrow`, `\Leftarrow`, `\Leftrightarrow` for implications
